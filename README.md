# Movie Scene Battle Analyzer

`movie_scene_battle_analyzer` crawls [Movie Scene Battles](https://moviescenebattles.blogspot.com), normalizes Blogspot posts into structured records, and publishes snapshot artifacts used by the hosted stats page.

## Purpose

This repository supports three engineering workflows:

1. **Data crawl**: pull posts from the Blogspot JSON feed.
2. **Snapshot build**: generate deployable artifacts under `data/`.
3. **Snapshot verification**: ensure `site_stats.json` exactly matches the current dataset.

No third-party Python dependencies are required.

## Architecture at a glance

```text
movie_scene_battle_analyzer/
  cli.py         # CLI options and command entrypoint
  crawler.py     # Feed pagination, parsing, and stat aggregation
  models.py      # BattlePost / SiteStats / CrawlDataset dataclasses
scripts/
  build_site_snapshot.py   # Rebuild dataset + stats payload
  verify_site_snapshot.py  # Validate schema and strict consistency
index.html                # Static page that renders data/site_stats.json
```

## Public interfaces

### CLI

```bash
python3 -m movie_scene_battle_analyzer [--max-posts N] [--include-content] [--output PATH]
```

Supported flags:

- `--max-posts` (default: `500`): maximum posts to crawl; must be `> 0`.
- `--include-content` (default: off): include full extracted post text in `content_text`.
- `--output` (default: `data/moviescenebattles_dataset.json`): dataset output path.

Example:

```bash
python3 -m movie_scene_battle_analyzer --max-posts 300 --output data/moviescenebattles_dataset.json
```

### Python API

```python
from movie_scene_battle_analyzer import crawl_moviescenebattles, save_dataset

dataset = crawl_moviescenebattles(max_posts=300, include_content=False)
save_dataset(dataset, "data/moviescenebattles_dataset.json")
```

`crawl_moviescenebattles` also supports:

- `page_size` (default `150`, must be `> 0`)
- `timeout` in seconds (default `30`)

## Data contracts

### `data/moviescenebattles_dataset.json`

Top-level keys:

- `site_title`
- `site_url`
- `posts` (array of `BattlePost`)
- `stats` (`SiteStats`)

`BattlePost` fields:

- `post_id`, `title`, `url`
- `published_at`, `updated_at` (ISO-like datetime string or `null`)
- `comment_count`, `categories`, `word_count`
- `content_text` (`null` unless `--include-content` is enabled)

`SiteStats` fields:

- `total_posts`, `total_comments`
- `average_comments_per_post`, `average_words_per_post`
- `posts_with_explicit_matchup`
- `posts_by_year`
- `top_categories` (top 10 by count)
- `most_commented_posts` (top 5 by comment count)
- `last_post_update`, `crawl_completed_at`

### `data/site_stats.json`

Generated by `scripts/build_site_snapshot.py` with:

- `site_title`
- `site_url`
- `generated_from_posts`
- `stats` (same object as `dataset["stats"]`)

`scripts/verify_site_snapshot.py` enforces strict equality between this payload and values derived from `data/moviescenebattles_dataset.json`.

## Operational runbook

### Rebuild snapshot artifacts

```bash
python3 scripts/build_site_snapshot.py
```

Current behavior of this script:

- crawls up to `1000` posts
- does **not** include full content text
- writes:
  - `data/moviescenebattles_dataset.json`
  - `data/site_stats.json`

### Verify artifacts are internally consistent

```bash
python3 scripts/verify_site_snapshot.py
```

This check fails if required keys are missing or if `site_stats.json` is not an exact projection of `moviescenebattles_dataset.json`.

### Serve stats page locally

`index.html` fetches `data/site_stats.json` at runtime, so use an HTTP server instead of opening the file directly:

```bash
python3 -m http.server 8000
```

Then open `http://localhost:8000/`.

## CI workflows

### Verify snapshot (`.github/workflows/verify-site-snapshot.yml`)

- Triggers: pull requests targeting `main`, plus manual dispatch.
- Runs: `python3 scripts/verify_site_snapshot.py`.
- Purpose: block merges when snapshot artifacts are missing or out-of-sync.

### Refresh snapshot (`.github/workflows/refresh-site-snapshot.yml`)

- Triggers: daily cron (`20 6 * * *`) and manual dispatch.
- Runs:
  1. `python3 scripts/build_site_snapshot.py`
  2. `python3 scripts/verify_site_snapshot.py`
  3. automated PR creation/update via `peter-evans/create-pull-request`.

## Troubleshooting

### `ValueError: max_posts must be greater than 0`

- Cause: invalid `--max-posts` (or API argument).
- Fix: pass a positive integer.

### `site_stats.json is out of sync with moviescenebattles_dataset.json`

- Cause: one artifact was modified without rebuilding both.
- Fix:

  ```bash
  python3 scripts/build_site_snapshot.py
  python3 scripts/verify_site_snapshot.py
  ```

### Stats page shows "Could not load stats snapshot."

- Common causes:
  - `data/site_stats.json` missing
  - page opened via `file://` instead of HTTP
  - malformed JSON payload
- Fix: rebuild artifacts and serve with a local HTTP server.
