# Movie Scene Battle Analyzer

`movie_scene_battle_analyzer` crawls [Movie Scene Battles](https://moviescenebattles.blogspot.com), normalizes feed entries into typed records, and builds aggregate stats for analytics and ranking workflows.

## Quick start

No third-party Python dependencies are required.

```bash
python3 -m movie_scene_battle_analyzer --max-posts 500 --output data/moviescenebattles_dataset.json
python3 scripts/verify_site_snapshot.py
```

To refresh both snapshot artifacts used by the stats page:

```bash
python3 scripts/build_site_snapshot.py
python3 scripts/verify_site_snapshot.py
```

## Public interfaces

### CLI

```bash
python3 -m movie_scene_battle_analyzer [--max-posts N] [--include-content] [--output PATH]
```

Options:

- `--max-posts` (default: `500`): max number of feed entries to crawl.
- `--include-content` (default: disabled): include extracted post body text in `content_text`.
- `--output` (default: `data/moviescenebattles_dataset.json`): output path for dataset JSON.

### Python API

```python
from movie_scene_battle_analyzer import crawl_moviescenebattles, save_dataset

dataset = crawl_moviescenebattles(
    max_posts=300,
    include_content=False,
    page_size=150,
    timeout=30,
)
save_dataset(dataset, "data/moviescenebattles_dataset.json")
```

`crawl_moviescenebattles(...)` behavior and constraints:

- Uses Blogspot pagination parameters: `start-index` and `max-results`.
- Stops when `max_posts` is reached or feed pages return no more entries.
- Raises `ValueError` when `max_posts <= 0` or `page_size <= 0`.
- `timeout` applies to each feed request.

## Data model and artifacts

### In-memory dataclasses

- `BattlePost`: normalized post data (ids, timestamps, title/url, categories, comment/word counts, optional `content_text`).
- `SiteStats`: aggregate metrics (`total_posts`, `total_comments`, per-year counts, top categories, most-commented posts, timing fields).
- `CrawlDataset`: root object containing site metadata, `posts`, and `stats`.

### Generated files

- `data/moviescenebattles_dataset.json`
  - canonical crawl dataset: `{site_title, site_url, posts, stats}`
- `data/site_stats.json`
  - dashboard payload: `{site_title, site_url, generated_from_posts, stats}`
  - must stay byte-for-byte aligned with dataset-derived payload generated by `scripts/build_site_snapshot.py`

## Workflow runbook

### Local refresh and validation

1. Rebuild artifacts:
   ```bash
   python3 scripts/build_site_snapshot.py
   ```
2. Validate artifact shape + consistency:
   ```bash
   python3 scripts/verify_site_snapshot.py
   ```
3. If dashboard behavior needs manual checking, serve the repository over HTTP and open `index.html`.
   - The page fetches `data/site_stats.json` with `fetch(...)`; opening the HTML directly from disk can fail in some browsers.

### CI automation

- `.github/workflows/verify-site-snapshot.yml`
  - Runs on pull requests targeting `main` and manual dispatch.
  - Executes `python3 scripts/verify_site_snapshot.py`.
- `.github/workflows/refresh-site-snapshot.yml`
  - Runs on schedule (`20 6 * * *`) and manual dispatch.
  - Rebuilds artifacts, verifies them, then opens/updates an automated PR via `peter-evans/create-pull-request`.

## Troubleshooting and common pitfalls

- **`site_stats.json is out of sync ...`**
  - Cause: artifact mismatch after editing data files manually or partial refresh.
  - Fix: run `python3 scripts/build_site_snapshot.py` and re-run verification.
- **Crawler exits with `ValueError`**
  - Cause: invalid `max_posts` or `page_size` value.
  - Fix: use values greater than zero.
- **Dashboard shows "Could not load stats snapshot."**
  - Cause: missing `data/site_stats.json`, invalid JSON, or non-HTTP page load context.
  - Fix: rebuild artifacts and serve files from a local web server.
- **`content_text` is `null` in dataset**
  - Cause: crawl ran without `--include-content` (default behavior).
  - Fix: rerun with `--include-content` if downstream processing requires post body text.
